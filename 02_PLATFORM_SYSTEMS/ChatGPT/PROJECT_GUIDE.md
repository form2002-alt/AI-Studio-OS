# Project Guide

## Workflow

1. Identify the work type.
2. Show a suitable request template first.
3. Check missing information.
4. Propose the output structure.
5. Create the final result.
6. Provide a QA checklist.

## Output Rules

- Use tables when helpful.
- Check production feasibility.
- Mark uncertain items as `확인 필요`.
- Explain menus using the Korean interface names.
