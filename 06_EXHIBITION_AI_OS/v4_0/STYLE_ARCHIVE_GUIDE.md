# 전시 시스템 v4.0 Style Archive Guide

Style Archive Engine은 사용자가 승인한 이미지·일러스트·렌더링·그래픽·영상 스타일을 자동 분석·분류·코드화하고 후속 작업에 재사용하기 위한 운영 규칙입니다.

---

## 1. 기본 목적

| 항목 | 내용 |
|---|---|
| 목적 | 마음에 드는 시각 결과물을 스타일 자산으로 저장 |
| 적용 대상 | 일러스트, 이미지, 렌더링, 그래픽, 영상 스타일 |
| 사용 방식 | 사용자가 “이 스타일 저장해줘”라고 말하면 자동 실행 |
| 목표 | 전시 전체의 시각 정체성 유지 |

---

## 2. 대표 호출어

- `이 스타일 저장해줘`
- `아카이브에 넣어줘`
- `이 그림체를 기준으로 해줘`
- `앞으로 이 스타일로 통일해줘`
- `이 스타일로 다음 이미지 만들어줘`
- `스타일이 맞는지 검수해줘`
- `현재 스타일 목록 정리해줘`

---

## 3. 자동 생성 항목

| 항목 | 설명 |
|---|---|
| 스타일명 | 이미지 성격에 맞게 자동 생성 |
| 스타일 코드 | `STYLE-[분야]-[표현방식]-[순번]` 형식 |
| 분야 분류 | 역사 / 문화 / 예술 / 과학 / 생활 / 현대 / 자연 / 어린이 / 렌더링 / 영상 등 |
| 기준 이미지 설명 | 이미지의 핵심 특징 요약 |
| 핵심 스타일 | 선, 색, 질감, 명암, 구도, 분위기 |
| 유지 요소 | 후속 작업에서도 고정할 요소 |
| 변경 가능 요소 | 주제, 대상, 소품, 배경, 장면 등 |
| 금지 요소 | 스타일을 흐트러뜨리는 요소 |
| 적용 가능 용도 | 패널, 패브릭, SNS, 영상, 렌더링 등 |
| 후속 프롬프트 문장 | 다음 이미지 제작에 바로 넣을 문장 |
| QA 기준 | 스타일 일치도 검수 기준 |

---

## 4. 스타일 코드 체계

```text
STYLE-[분야]-[표현방식]-[순번]
```

## 예시

| 코드 | 의미 |
|---|---|
| STYLE-HISTORY-WASH-001 | 역사 전시용 담채 수채화풍 |
| STYLE-NATURE-HANDDRAWN-001 | 자연 전시용 손그림 삽화풍 |
| STYLE-SCI-DIAGRAM-001 | 과학 전시용 다이어그램풍 |
| STYLE-CULTURE-WATERCOLOR-001 | 문화행사용 감성 수채화풍 |
| STYLE-MODERN-GRAPHIC-001 | 현대 홍보용 그래픽풍 |
| STYLE-KIDS-CHARACTER-001 | 어린이 체험용 캐릭터풍 |
| STYLE-RENDER-REALISTIC-001 | 실사 렌더링 기준 스타일 |
| STYLE-SILHOUETTE-BLACK-001 | 레이저 컷팅용 흑백 실루엣 스타일 |

## 5. 분야 자동 분류 기준

| 분야 | 분류 기준 |
|---|---|
| 역사 | 시대, 사건, 인물, 유물, 복식, 생활사 중심 |
| 문화 | 지역문화, 축제, 전통, 생활문화, 무형문화 중심 |
| 예술 | 작가, 작품, 전시 브랜딩, 감상 경험 중심 |
| 과학 | 원리, 구조, 실험, 체험, 다이어그램 중심 |
| 생활 | 의식주, 직업, 일상 도구, 교육 이미지 중심 |
| 현대 | 도시, 정책, 기업, 브랜드, 홍보 중심 |
| 자연 | 동물, 식물, 생태, 환경, 지질, 기후 중심 |
| 어린이 | 체험관, 미션, 캐릭터, 놀이형 콘텐츠 중심 |
| 렌더링 | 공간, 건축, 제품, 집기, 실사 이미지 중심 |
| 영상 | 컷신, 모션그래픽, 미디어아트, 전이공간 중심 |

## 7. 스타일 일치도 QA 기준

| 검수 항목 | 기준 |
|---|---|
| 선 | 기준 이미지와 선 굵기·강약이 비슷한가 |
| 색 | 팔레트와 채도가 유지되는가 |
| 질감 | 수채·목탄·벡터·렌더링 질감이 유지되는가 |
| 명암 | 밝기와 대비가 기준과 맞는가 |
| 구도 | 여백과 배치 방식이 맞는가 |
| 대상 표현 | 인물·동물·사물 표현 방식이 같은 계열인가 |
| 정확성 | 주제의 사실 관계나 형태가 맞는가 |
| 금지 요소 | 금지한 표현이 들어가지 않았는가 |
| 제작성 | 실제 전시 출력·영상·가공에 적합한가 |
| 최종 판정 | 승인 / 재생성 / 폐기 |

## 8. 파일명 규칙

승인 이미지는 아래 구조로 저장합니다.

```text
[스타일코드]_[프로젝트명]_[대상]_[용도]_[승인일]
```

예시:

```text
STYLE-NATURE-HANDDRAWN-001_자연사전시_멸종위기동물_패널_20260706
STYLE-HISTORY-WASH-001_생활사전시_농부인물_패브릭_20260706
STYLE-SCI-DIAGRAM-001_과학전시_에너지흐름도_인포그래픽_20260706
```
