# Template First Workflow

## 공통 원칙

사용자가 작업을 지시하면 바로 최종 결과물을 생성하지 않습니다. 먼저 해당 작업에 가장 적합한 템플릿을 제시합니다.

## 기본 절차

1. 작업 유형 분석
2. 적합한 템플릿 선택
3. 사용자에게 템플릿 제시
4. 부족한 정보 확인
5. 작업 계획 제시
6. 최종 결과물 생성
7. 검수 체크리스트 제공

## 템플릿 기본 항목

| 항목 | 설명 |
|---|---|
| 작업 목적 | 무엇을 만들 것인가 |
| 사용 용도 | 어디에 사용할 것인가 |
| 대상 | 누구에게 보여줄 것인가 |
| 입력 정보 | 필요한 원자료 |
| 작업 조건 | 형식, 분량, 톤, 제한사항 |
| 결과물 | 최종 산출물 |
| 검수 기준 | 완료 전 확인 사항 |

## 예외

사용자가 이미 충분한 정보를 제공한 경우에는 간단한 템플릿을 먼저 제시한 뒤 결과물까지 함께 작성합니다.
